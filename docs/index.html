Die produktive App trennt Bedienung, Administration und Betrieb sauber voneinander.

Sicherheit

Die oeffentliche Seite ist statisch. Die App bleibt intern.

GitHub Pages veroeffentlicht nur diese Informationsseite. Die echte Wartungs-App laeuft intern, damit Login, SQLite-Datenbank, Sessions und Admin-Funktionen nicht oeffentlich im Browser liegen.

PBKDF2 Passwort-Hashing CSRF-Schutz fuer Schreibaktionen Login-Rate-Limit HttpOnly Sessions Audit-Log fuer Aenderungen
Lokaler Start python .\run.py Windows Autostart .\scripts\install-scheduled-task.ps1 Optional HTTPS .\scripts\start-https.ps1

Betrieb

Produktiv auf internem Rechner oder Server starten.

Die App kann auf einem Windows-PC im Netzwerk laufen. Fuer dauerhaften Betrieb ist ein Scheduled Task oder ein Windows-Service sinnvoll. Fuer Netzwerkzugriff sollte HTTPS ueber Zertifikat oder Reverse Proxy genutzt werden.
